# Braillinator
(v1 cloned from GitLab)

## Description
The Braillinator is a text from image to Braille converter that scans text using a phone camera and translates to Braille. The translation is done by pistons that automatically push out of a corresponding physical device, where the user can place their fingers on. These pistons push in and out, spelling out the text in Braille, letter by letter, giving the same tactile experience as Braille. This physical device also has play/pause, speed adjustments, and other controls.

## v2: OCR Enhancement

### The Problem
The original pipeline relies on Tesseract OCR, which performs poorly on blurry or low-quality phone camera images. Since Braillinator is designed for visually impaired users, OCR errors directly result in incorrect Braille output — making reliability critical.

### Approach
We implemented a small conditional U-Net trained from scratch. The model takes two inputs: the blurry image and a per-word OCR confidence heatmap generated by Tesseract's first pass. A Tesseract character prediction is also encoded and injected as a conditioning signal via FiLM (feature-wise linear modulation) at every decoder stage, so the model knows roughly what text should be present when reconstructing the image. One forward pass produces a sharpened image, which is then passed to Tesseract for a final, more accurate read.

The model is under 50MB and fast enough for inference on a Raspberry Pi 4.

Training used the SmartDoc ICDAR 2015 dataset — 3,630 real smartphone-captured document photos — degraded synthetically (gaussian blur, motion blur, JPEG compression, noise) to create 16,000 paired training samples. The loss is pixel-level (MultiScaleL1 + gradient penalty) against the original unblurred phone photo.

The quality gate runs enhancement only when Tesseract detects ≥10 words and the miss rate exceeds 25% — images that are already readable are passed through unchanged.

### Results
Best results on the two problem images (val loss 0.23 → 0.12 after 60 epochs):
- TestImage_1 (heavily blurred): 85.7% → 39.3% miss rate (−46.4%), word count preserved
- TestImage_4: 30.6% → 38.5%, but visually readable — no noise generation

Average effective miss rate delta across all 16 test images: −2.4%. Enhanced outputs are readable text; earlier training runs produced grid/noise patterns that are now resolved.

Full training history in [v2_progress_log.md](v2_progress_log.md).
